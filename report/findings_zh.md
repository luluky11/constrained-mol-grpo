# 思维链(CoT)对约束分子生成的影响

> 主线一句话：用"GRPO + RDKit 可验证奖励"训练小模型(Qwen2.5-1.5B)做约束分子生成，**系统对比不同思维链(CoT)风格的效果**，并与数学任务、难度变化做对照。核心发现：**在该任务上，轻量 CoT(none/short/budget)与无 CoT 基本持平，而冗长/自然语言 CoT(structured/constructive/gemini)明确有害；这与 CoT 在数学上的明显增益形成鲜明对比。** 全部对照在同一标准库(library_v2)上完成。

---

## 0. TL;DR

1. **训练大幅有效，且 SFT 与 GRPO 协同**：未微调 base 几乎不会做(greedy 1.5%)；SFT-only 0.052、GRPO-only 0.077 单独都弱，**SFT+GRPO 合起来 0.302**(远超两者之和)——缺一不可(详见 §3.5 2×2 消融)。
2. **主线：轻量 CoT 与无 CoT 打平，重型 CoT 有害**。none/short/budget 三者基本并列在顶部(差距在单种子噪声内)；constructive→structured→gemini 随推理变"重"单调下滑，gemini 最差且合法率下降。即 **CoT 在本任务最多与"不写推理"持平，冗长推理反而拖累。**
3. **跨域对照**：同样的 CoT 在**数学(GSM8K)上明显有用**(29.2% vs 直接 4.0%)，在分子生成上无用——**说明 CoT 的价值高度依赖任务类型**。
4. **难度对照(诚实结果)**：在同一库内单纯改变约束难度，**未观察到稳健的、随难度变化的 CoT 效应**(差值落在种子噪声内)。即 CoT 的"无用"不是某个难度点的偶然。

---

## 1. 任务与方法

- **任务**：给定性质约束(MolWt/logP/QED/TPSA/HBD/HBA/可旋转键/环数中 1–3 个)+ 结构约束(0–3 个官能团 SMARTS)，生成**一个**同时满足全部约束的合法分子(SMILES)。
- **可验证奖励(RLVR)**：约束是否满足由 RDKit 程序化判定，无需人工标注；答案不唯一(任何满足约束的分子都算对)。
- **GRPO**(TRL)：对同一 prompt 采样多个输出，用组内相对奖励更新策略。奖励含格式、合法性、性质满足(连续打分)、结构匹配、类药性、全满足 bonus、多样性、简洁性。
- **模型/训练**：Qwen2.5-1.5B-Instruct + LoRA(r=16)；先 SFT 热身(教格式+合法分子+对应 CoT 风格的推理)，再 GRPO。
- **CoT 风格**(SFT 阶段注入的 `<reasoning>` 内容)：
  - `none`：直接给答案(无推理)；
  - `short`：泛化的一句话；
  - `structured`：分步模板(RDKit 数值锚定)；
  - `constructive`：化学构造式(骨架+片段)；
  - `budget`：按基团贡献做数值预算；
  - `gemini`：从 Google Gemini 蒸馏的高质量自然语言推理。

---

## 2. 评测协议

- `all_ok`(greedy)：贪心解码下全部约束同时满足的比例(≈pass@1)。
- `pass@8`：每条 prompt 采样 8 个，至少一个全满足的比例。
- `valid`：RDKit 可解析比例。
- 规模：greedy n=1000；pass@8 n=300、每条 8 采样；种子固定。

---

## 3. 主结果

### 3.1 训练 vs 未训练

| 模型 | greedy all_ok | pass@8 | valid |
|---|---|---|---|
| base-1.5B(未微调) | 0.015 | 0.083 | 0.17 |
| SFT+GRPO(none) | 0.302 | 0.570 | 0.976 |

→ 训练让 greedy 提升约 **20×**、合法率到 ~0.98。**GRPO+可验证奖励确实学到了"按约束生成合法分子"。**

### 3.2 主线：各类 CoT 风格对照(1.5B, library_v2 单库)

所有风格同库(v2)、同训练/评测配方(NS=4000, gen=8, maxlen=192)，SFT 用**同一份种子文件**(同批分子对，只差推理文本)，仅 reasoning 风格不同。

| reasoning 风格 | greedy all_ok | pass@8 | valid |
|---|---|---|---|
| **budget(数值预算)** | **0.317** | **0.577** | 0.986 |
| short | 0.309 | 0.527 | 0.987 |
| none(无 CoT) | 0.302 | 0.570 | 0.976 |
| constructive(构造式) | 0.286 | 0.503 | 0.983 |
| structured(分步模板) | 0.263 | 0.487 | 0.978 |
| gemini(高质量自然语言) | 0.212 | 0.463 | 0.900 |

**结论(两层)**：
1. **顶部 none / short / budget 基本打平**(greedy 0.30–0.32、pass@8 0.53–0.58，差距落在单种子噪声 ±0.02–0.03 内)。budget 名义最高但**不显著** → **轻量 CoT 既不明显帮、也不明显坑**。
2. **越冗长越差，单调下滑**：constructive → structured → **gemini 最差**(greedy 0.212，且 **valid 由 ~0.98 掉到 0.90**)。**长推理把最终 SMILES 的合法生成挤掉了。**

合起来：**在约束分子生成上，CoT 最多与"不写推理"持平；越接近自然语言、越长的推理越有害。** 与下文数学任务形成鲜明对比。

### 3.3 跨域对照：CoT 在数学上有用吗?(GSM8K, 0.5B base)

| 提示 | 准确率(500 题) |
|---|---|
| **CoT(含推理)** | **0.292** |
| 直接答案 | 0.040 |

→ **同样的 CoT，在数学上是 7× 的巨大增益。** 与分子任务(CoT 至多与 none 持平、冗长反而有害)形成鲜明对比，说明**CoT 的价值高度依赖任务类型**：数学需要可串联的多步推导，CoT 帮得上；约束分子生成更像"一步到位的受约束采样"，长推理反成负担。

### 3.4 难度对照(诚实结果)：CoT 的效果会随难度变化吗?

在同一库(v2)内，**只改变约束精度(收紧性质窗口 τ)**，其余(出题、奖励、训练步数)全不变，比较 none vs budget 在四档难度下的差值：

| 难度档 | 解密度 | Δgreedy(bud−none) | Δpass@8(bud−none) |
|---|---|---|---|
| 易(τ1.2) | ~0.25 | −0.008 | +0.013 |
| 中(τ0.7) | ~0.067 | +0.035 | +0.033 |
| 较难(τ0.5) | ~0.030 | +0.001 | +0.067 |
| 很难(τ0.35) | ~0.0096 | −0.022 | −0.047 |

**诚实结论**：差值幅度(±0.01–0.07)与种子噪声(常 ±0.02–0.03)同量级，且为单种子——**未观察到稳健的、随难度单调变化的 CoT 效应**。即"CoT 在本任务无用"并非某个难度点的偶然，而在各难度大致一致。
（附带方法学含义：跨库时看到的"难分布上 CoT 略好"现象，在单库内独立控制难度后并未复现，提示那更多源于**分布/化学型差异**而非难度本身。）

> **注**：本节(§3.4)是独立子研究——8 格内部训练/评测配方完全一致、GRPO 设置与 §3.2 相同；但其 **SFT 配方与 §3.2 不同**(§3.4 为库采样 6000/epochs1/max_len512；§3.2 为共享种子文件/epochs2/max_len640)。因此 §3.4 的 **Δ(none vs budget 差值)有效**，但**其绝对数不应与 §3.2 直接横比**。

### 3.5 SFT 与 GRPO 的分工（及 LoRA 的位置）

**两个阶段各司其职：**
- **SFT(监督热身)= 让模型"按格式起步"**：用真实分子的(prompt→reasoning+SMILES)对做模仿，学会 `<reasoning>/<answer>` 格式与"大致像分子的串"及 CoT 风格。但它只模仿示例，**既不直接逼模型满足当前约束、也不保证合法**(见下表 SFT-only valid 仅 0.34)。
- **GRPO(强化)= 真正的主力**：以 RDKit 为可验证奖励，对采样输出按合法性/性质/结构/全满足打分并相对更新——**合法率和约束满足几乎都是在这一步被拉上去的**。

一句话：**SFT 只负责"起步引导"，GRPO 才把"写得合法 + 写得对"做出来。**

**2×2 消融(全部 v2、同评测协议、同 GRPO 预算 1000 步、SFT 与 §3.2 同配方)**：

| | 不做 GRPO | + GRPO |
|---|---|---|
| **不做 SFT** | base：0.015 / 0.083 / 0.171 | GRPO-only：0.077 / 0.217 / 0.761 |
| **做 SFT(none)** | SFT-only：0.052 / 0.263 / 0.344 | **SFT+GRPO：0.302 / 0.570 / 0.976** |

(每格 = all_ok / pass@8 / valid)

→ **核心发现：SFT 与 GRPO 是协同(超加性)关系，缺一不可。**
- **单独都很弱**：SFT-only 0.052、GRPO-only 0.077；
- **合起来 0.302，远超两者之和(0.13)** —— SFT 提供"格式 + 合法分子"先验，GRPO 才能在此先验上把约束满足真正优化出来。
- **同样 1000 步 GRPO 预算下，没有 SFT 热身 all_ok 只有 0.077(约 1/4)**：GRPO-only 虽能把 reward/合法率(→0.76)爬起来，却**始终够不到约束满足**——印证 §3.4 之外的另一点：**SFT 的初始化在本任务上价值巨大，不可省。**

**为什么本任务需要 SFT，而原 GSM8K 项目不用？**
- 原数学项目**直接在 Qwen-Instruct 上做 GRPO，无 SFT**——因为 base 本就会按格式做数学题(base CoT 已 29%)，起点不差。
- 本任务 base **几乎不会**(all_ok 0.015)。实测 GRPO-only 并非"学不动"——它能把合法率爬到 0.76、reward 也起来了——但**冷启动慢、且同预算下 all_ok 止步 0.077(SFT+GRPO 的 ~1/4)**。说明**SFT 的格式/合法先验是 GRPO 高效优化约束满足的前提**，二者协同；省掉 SFT 不是"完全不行"，而是"同等预算下显著更差"。

**LoRA 的位置**：LoRA 不是独立阶段，而是 **SFT 与 GRPO 两个阶段都使用的参数高效微调方法**——冻结底模、只训低秩适配矩阵(r=16)，每阶段结束合并回底模。流水线：`base →[SFT(LoRA)]→合并→[GRPO(LoRA)]→合并→评测`。

---

## 4. 结论

- **CoT 的有效性强依赖任务**：数学(可串联推导)明显受益(29% vs 4%)；约束分子生成(受约束采样)上 **CoT 至多与 none 持平**。
- 在分子任务上呈现清晰的**"越轻越好"梯度**：none/short/budget 并列居首，constructive→structured→**gemini 单调变差**——**推理越长/越自然语言，越拖累最终分子的合法生成**(gemini valid 跌至 0.90)。
- 单库内独立控制难度，**CoT 的(无)效应在各难度大致一致**，无稳健难度依赖；跨库看到的"难处 CoT 略好"未在单库复现，提示其源于分布差异而非难度。

## 5. 局限与下一步

- 主对照为单种子；顶部 none/short/budget 的并列、以及难度差值，幅度都在噪声量级，**要做成强结论需每条件 2–3 个种子加误差棒**。
- 模型止于 1.5B；CoT 是否随模型增大改观，可加一个更大模型的点验证。
- "重型 CoT 有害"的机制(长文本挤占合法生成)可进一步用"限定推理长度/截断"实验直接验证。
- ~~统一条件的 SFT-only 消融~~ **已完成**(见 §3.5 三阶段消融表，v2 同设置)：证实 GRPO 是合法性与约束满足的主力来源。
